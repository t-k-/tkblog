Wildcard expansion (SLT)


  • case 1: \(\quad \color{red}{\text{?1}} \quad\) matching \(\quad \color{red}{a + b + \dfrac x y}\)
  • case 2: \(\quad \color{red}{\text{?1}}^2 \quad\) not matching \(\quad {a + b + \dfrac x y}\)
  • case 3: \(\quad \color{red}{\text{?1}} \quad\) matching \(\quad \color{red}{a^2 + b + \dfrac x y}\)
  • case 4: \(\quad \color{red}{\text{?1}}^2 \quad\) matching \(\quad \color{red}a^2 + b + \dfrac x y\)


Aligning: (\(\color{red}{\text{?1}}\), ) with (V!a, )

Wildcard expansion (SLT)


In case 1 and 3, we say wildcard is horizontally expandable


But it may stop expanding when the next node of query is the same or unifiable with the next expanding node

  • case 1: \(\quad \color{red}{\text{?1}} + ... \quad\) matching \(\quad \color{red}{a} + b + \dfrac x y\)
  • case 3: \(\quad \color{red}{\text{?1}} + ... \quad\) matching \(\quad \color{red}{a^2} + b + \dfrac x y\)

Wildcard expansion (OPT)


In OPT case, wildcard expansion is simply matching the entire subtree

Possible improvements



  • 1. To extend alignment algorithm beyond top-K, OPT permutation complexity is not acceptable. (OPT handles commutative nodes well when search from leaf to root)

  • 2. SLT Wildcard greedy \(\alpha\)-equivalence may cause unexpected result.

  • 3. Core engine is not doing unified match.
